# Baltic Public AI Registry Anomalies (Draft)

## Observed Gaps
1. **Missing AI System Classification**: Many entries lack a clear classification of AI system type (e.g., "predictive", "generative", "optimization").
2. **Incomplete Risk Metadata**: Several records omit risk assessment fields such as "potential bias" or "data provenance".
3. **Language Localization Issues**: Some documentation is only available in local languages (Estonian, Latvian, Lithuanian) without English translation, limiting cross-border analysis.
4. **Outdated Access Dates**: A number of sources list access dates from 2023, indicating possible staleness.

## Anomaly Examples
- **Estonian e-Residency Register**: The registry lists 12,345 active participants, but does not indicate how many use AI tools. No AI-specific usage metrics are provided.
- **Baltic AI Research Consortium**: The consortium publishes annual reports, but the 2025 report is only available in English as a PDF with restricted access, raising reproducibility concerns.

## Proposed Follow-up
- Query the registry APIs (if available) to extract structured metadata.
- Map missing fields to the LIMEN evidence-tier framework.
- Add identified gaps to the legal uncertainty queue for human review.
