# Enforcement Action Taxonomy

This taxonomy provides standardized categories for regulatory actions related to AI governance. It is designed for cross-jurisdictional comparability while preserving original source meaning.

## Core Categories

1. **Corrective Measures**
   - `warning`: Formal notices requiring future compliance
   - `remediation_order`: Mandatory actions to fix violations

2. **Financial Penalties**
   - `fine`: Monetary penalties imposed
   - `recovery_payment`: Restitution payments ordered

3. **Operational Restrictions**
   - `serviceSuspension`: Temporary shutdown of services
   - `licensure_revoke`: Removal of operating permissions

4. **Preventative Actions**
   - `guidance`: Non-binding regulatory advice
   - `compliance_review`: Mandatory audits without immediate penalties

## Mapping Guidance

When assigning categories:
1. Start with literal source description
2. Map to nearest core category while preserving original meaning
3. Add jurisdiction-specific notes in `additional_info` field
4. Flag ambiguous cases with `uncertainty_score` (1-5)
